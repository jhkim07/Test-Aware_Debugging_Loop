# 마크다운 파일 명명 규칙

## 규칙

모든 마크다운 파일은 일련번호를 앞에 붙여서 관리합니다.

## 번호 체계

- `00-09`: 핵심 프로젝트 문서
  - 00: README (프로젝트 기본)
  - 01: PROJECT_ANALYSIS (프로젝트 구조)
  - 02: REQUIREMENTS_ANALYSIS (요구사항)
  - 03: SPLIT_MECHANISM_EXPLANATION (핵심 메커니즘)
  - 04: COMPLETION_SUMMARY (구현 완료)
  - 05: FINAL_COMPARISON_REPORT (실험 결과)
  - 06: TESTING_MODULES_AND_FLOW (테스팅 모듈)
  - 07: CODEBASE_LOCATION_AND_EXECUTION (코드베이스 위치)
  - 08: DOCUMENTATION_INDEX (문서 인덱스)
  - 09: RUNNING_EXPERIMENTS (실험 실행)

- `10-19`: 디버깅 및 패치 관련
  - 10: PATCH_DEBUG_STATUS
  - 11: PATCH_DEBUG_SUMMARY
  - 12: PATCH_FIX_SUMMARY
  - 13: PATCH_FIX_FINAL

- `20-29`: 실험 결과 및 분석 (예약)

- `30-39`: 기타 기술 문서 (예약)

## 새 문서 추가 시

1. 적절한 번호 범위 선택
2. 다음 사용 가능한 번호 확인
3. `NN-DESCRIPTIVE_NAME.md` 형식으로 파일명 작성
4. `08-DOCUMENTATION_INDEX.md`에 추가

## 예시

```bash
# 올바른 형식
14-NEW_FEATURE_DOCUMENTATION.md
15-ANOTHER_ANALYSIS.md

# 잘못된 형식
NEW_FEATURE.md  # 번호 없음
14_new_feature.md  # 소문자 사용
```
